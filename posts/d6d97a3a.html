HPA* 算法

Created2025-01-17|Updated2025-01-23|任务规划单机规划
|Word Count:2.3k|Reading Time:7mins

HPA*（Hierarchical Path-Finding A*）是一种针对大规模网格地图的分层路径规划算法，由 Botea 等人于 2004 年提出。它通过将地图划分成多个区域（clusters），并在更高层级（抽象层）进行搜索，从而加速在大地图上的路径搜索过程。下文先对 HPA* 的主要思想和流程进行介绍，然后给出一个简化的 Python 伪实现示例，用于帮助理解。

一、HPA* 的核心思想

  1. 地图分块（Clustering）
    • 将原始大网格地图分割成若干个小块（clusters），如每个小块为 \(C \times C\) 的网格。
    • 通过分块减少需要显式搜索的节点数量，从而加快整体搜索。
  2. 抽象图构建（Abstraction）
    • 在每个小块中，选取若干 “出入口（entry/exit points）” 做为该块对外连接的入口/出口。
    • 这些出入口间在小块内部使用 A* 或类似方法预先计算最短子路径，形成一个抽象节点与边的图。
    • 在更高层次的抽象图中，每个小块视为一个抽象节点，块与块之间通过出入口点相连。
  3. 分层搜索
    1. 高层搜索：在抽象图上进行 A* 等基于图的搜索，找到从起始块到目标块的抽象路径。
    2. 低层修正：对抽象路径上的每个块，再调用本地的子路径（已预先计算或实时搜索）来拼接出完整、精细化的路径。
    3. 这样就不需要在原图上做全局搜索，而是在抽象层快速定位大体方向，在局部小块内完成细化路径。
  4. 动态更新
    • 如果地图发生变化（障碍物增减），可局部地更新受影响的小块抽象，而不必重新计算整张地图。
    • 能适应动态环境，效率比普通 A* 在大地图中高得多。

优点

  • 缩小搜索空间：只需在抽象层搜索，以及在局部块内搜索，减少搜索节点总量。
  • 可扩展：对大规模地图有良好效果，并可进行多级抽象（HPA*、Hierarchical HPA*）。

缺点

  • 预处理：需要花费额外时间和内存构建抽象图/出入口点信息；初次路径可能不如 A* 直接搜索快，但在多次查询时通常更优。
  • 近似最优：由于抽象，可能导致路径比精确最优路径稍长。

二、HPA* 算法流程示例

  1. 分块
    • 将网格划分为大小为 \(C \times C\) 的若干块。
    • 对每个块，标识其内部所有可行点（空地）以及与相邻块连通的 “出入口点（entry/exit points）” 。
  2. 构建抽象图
    • 对于每个块：
      • 在块内部预先计算任意出入口点之间的最短路径（可用 A*）。
      • 这些出入口点在抽象层成为抽象节点，出入口点之间的最短路径距离成为抽象边的代价。
    • 在块与块之间，若有共享出入口点或相邻出入口点，则在抽象图中建立边连接。
  3. 高层搜索
    • 给定起点和终点，找到它们各自所在的块和对应出入口点。
    • 在抽象图中使用 A*（或 Dijkstra 等）搜索，从起点所在块的出入口点到终点所在块的出入口点，得到抽象路径（sequence of blocks and doorways）。
  4. 低层拼接
    • 对抽象路径中的相邻块之间，通过预先存储的出入口点-出入口点最短路径或直接在块内部用 A* 进行局部搜索，拼接成完整精细的网格路径。

三、HPA* 简化 Python 实现（示例伪代码）

下述示例代码仅演示主要结构，忽略了很多实现细节，比如网格如何准确分块、如何选择出入口点等。只用来帮助理解 HPA* 的基本流程。

代码思路：

  1. 分块：build_clusters 将 64×64 地图按 8×8 区域划分成 (64/8) * (64/8) = 8*8 = 64 个块，每个块记录其行列范围和唯一 ID。

  2. 出入口点：

    • 在每个簇边界上查找与相邻簇连通的点（ “出入口” ）。
    • 对这些出入口点进行簇内 A* 或 BFS 计算其两两最短子路径，储存抽象图信息。

  3. 抽象图：

    • 把每个簇作为一个抽象节点，在抽象层将不同簇之间的出入口点互连并存储距离。
    • high_level_search 在抽象层对簇进行 A*，返回抽象路径（簇序列 + 出入口点序列）。

  4. 低层修正：

    • refine_path 根据抽象路径中相邻的簇与出入口点信息，进行局部搜索或查表，拼接出完整网格级别路径。

因为 HPA* 的实现细节较繁琐（尤其是出入口点搜索、抽象图构建、局部搜索拼接等），上面仅演示框架与关键函数，具体可参考 Botea 2004 年的相关论文或开源项目进行深度实现。

三、关键要点

  1. 分层思想：大地图被切分为多个簇，局部搜索在簇内得到出入口点间的最短路径，抽象层搜索只在簇与簇之间的连通进行快速规划。

  2. 出入口点与抽象边：

    • 需在簇边缘上找到空地点，可以作为出入口点来与相邻簇相连。
    • 计算出入口点间的最短路径，需要在簇内部做一次 BFS/A* 并记录结果。

  3. 多次查询的效率：

    • 若在同一个大地图上多次进行不同起终点的规划，HPA* 的预处理（出入口点-出入口点最短路）可被复用，极大提升查询效率。

  4. 动态更新：

    • 如果地图发生变化（障碍增删），只需更新受影响簇的内部最短路径，不必整体重算。

四、优缺点

  • 优点：适合大地图，可以显著减少在精细网格上全局搜索的开销；在多次路径查询时效率极大提升。
  • 缺点：需要额外的预处理；当地图障碍物分布复杂时，出入口点选择和局部搜索也不简单；生成的路径有时不如全局精细搜索的路径短，但通常误差很小。

参考文献

  • Botea, A., Müller, M., & Schaeffer, J. (2004). Near Optimal Hierarchical Path-Finding. Journal of Game Development, 1(1):7-28.
  • Botea, A., Müller, M., & Schaeffer, J. (2004). Using abstraction for planning in real-time strategy games. AIIDE.
